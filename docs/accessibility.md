# Accessibility Features

djust-theming ships with built-in support for reduced motion preferences,
high contrast mode, and print optimization. These features require **no
configuration** -- they activate automatically based on the user's operating
system or browser settings.

## Reduced Motion

When a user has enabled "Reduce motion" in their OS accessibility settings,
djust-theming responds via `@media (prefers-reduced-motion: reduce)`:

- All duration tokens (`--duration-fast`, `--duration-normal`, `--duration-slow`,
  `--duration-slower`) are set to `0ms`.
- `animation-duration` and `transition-duration` are forced to `0.01ms`.
- `animation-iteration-count` is forced to `1`.
- `scroll-behavior` is set to `auto` (disables smooth scrolling).

These rules are generated by `get_reduced_motion_css()` in
`djust_theming.design_tokens` and included automatically in the CSS output
of `generate_design_tokens_css()` and `generate_design_tokens_classes_css()`.

### Testing locally

In Chrome DevTools, open the **Rendering** tab and set
*Emulate CSS media feature prefers-reduced-motion* to `reduce`.

## High Contrast

When a user has enabled increased contrast in their OS or browser,
djust-theming responds via `@media (prefers-contrast: more)`:

- `--border-width` is set to `2px`.
- `--ring-width` is set to `3px` and `--ring-offset` to `3px`, making focus
  indicators larger and more visible.
- All elements receive `border-style: solid` to ensure borders are always
  rendered.
- `:focus-visible` outlines use the ring tokens for consistent, high-visibility
  focus indicators.
- Interactive components (`.btn`, `.card-theme`, `.input-theme`, badges) receive
  increased `border-width`.

These rules are generated by `get_high_contrast_css()` in
`djust_theming.design_tokens`.

### Testing locally

In Chrome DevTools, open the **Rendering** tab and set
*Emulate CSS media feature prefers-contrast* to `more`.

## Print Stylesheet

A dedicated print stylesheet is included automatically via `theme_head.html`
with `media="print"`, so it has **zero impact on screen rendering performance**.

The stylesheet (`djust_theming/static/djust_theming/css/print.css`) applies the
following optimizations when a page is printed or saved as PDF:

### Hidden elements

Interactive and navigation elements are hidden:

- `.sidebar`, `.navbar`, `.theme-switcher`, `.theme-mode-btn`
- `.theme-preset-select`, `.toast`, `.modal-overlay`
- `button[type="button"]`, sidebar toggle, floating elements

### Readable colors

- Body background forced to white, text forced to black.
- Links are black with underline decoration.
- Headings are forced to black.
- Muted text uses `#333` for adequate contrast.

### Visual effects removed

- `box-shadow`, `text-shadow`, animations, and transitions are removed from all
  elements.
- Cards receive a simple `1px solid #ccc` border on white background.

### Link URLs displayed

External link URLs are appended after the link text:

```
Visit our docs (https://example.com/docs)
```

Fragment (`#`) and `javascript:` links are excluded.

### Page break control

- Headings (`h2`, `h3`) avoid page breaks after them.
- Images, SVGs, cards, tables, `pre`, and `blockquote` elements avoid page
  breaks inside them.

### Layout adjustments

Main content areas (`.main-content`, `.layout-content`,
`.layout-centered-content`) expand to full width with no margin or padding.

### Testing locally

Use **Ctrl+P** (or **Cmd+P** on macOS) to open the print preview, or in Chrome
DevTools open the **Rendering** tab and set *Emulate CSS media type* to `print`.
