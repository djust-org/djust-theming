### Design Decisions
1. Slots implemented as optional context variables (not Django {% block %} tags) because component templates are standalone (not extended via {% extends %})
2. Slot variables use |safe filter to allow HTML content — this is intentional since slots are meant to contain rendered HTML fragments
3. When no slot variable is provided, templates produce identical output to before — full backward compatibility
4. Contracts use frozen dataclasses for immutability and hashability
5. ComponentTestCase uses stdlib html.parser for zero additional dependencies
